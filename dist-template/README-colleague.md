# chexian-query — 同事使用手册

> 给业务同事:用 TRAE + GLM-5.1 自然语言查车险数据,无需写代码、无需联外网。

---

## 我能问什么样的问题?

直接说中文,GLM 帮你写 SQL 跑出结果。例:

- "2026 年 4 月各三级机构的剔摩满期赔付率,从高到低排"
- "营业货车按吨位段看赔付率和保费规模,哪段在亏"
- "本月驾意险渗透率,按机构看,哪些机构主全没卖驾意"
- "Top30 业务员标保排名,带同比"

更多样板:`docs/examples/`(5 个)

## 5 步上手(只需一次)

### 第 1 步:确认装好了基础软件

需要 IT 帮装(找 IT 时把这一段截图给他):
- **Node.js 22 LTS**(必须,跑数据查询用)
- **TRAE**(必须,AI 编辑器)
- **TRAE 里配好 GLM-5.1 API Key**(自付或公司统一,不在本项目范围)

验证装好了:打开 PowerShell,跑一句:
```powershell
node --version
```
应输出 `v22.x.x` 或更高。如果报"找不到 node",找 IT 装。

### 第 2 步:确认能访问数据共享盘

打开"此电脑",在地址栏输:
```
\\10.120.0.87\chexian-query
```

第一次访问可能让你输用户名/密码 — 找数据负责人要,勾选"记住凭据"。

如果连不上 → 找 IT 检查内网 / 找数据负责人确认中转机是否开机。

### 第 3 步:解压 chexian-query 到 D 盘

数据负责人会给你一个 `chexian-query-YYYYMMDD.zip`(可能放在群里 / 邮件 / `\\10.120.0.87\chexian-query\release\`)。

把它解压到 **`D:\chexian-query`**(右键 → 解压到当前文件夹,或用 7-Zip)。

⚠️ **必须放在 `D:\chexian-query`**(不是 `D:\下载\chexian-query`,不是桌面)— 这个统一路径方便排错。

### 第 4 步:用 TRAE 打开

启动 TRAE → 点"打开文件夹" → 选 `D:\chexian-query` → 确认。

TRAE 会自动加载 `AGENTS.md` 和 `CLAUDE.md`(GLM-5.1 的行为指南)。

### 第 5 步:试一个问题

在 TRAE 的对话框里输:

```
帮我看 2026 年 4 月各三级机构的剔摩满期赔付率,从高到低排
```

GLM 会:
1. 读 `docs/business-rules.md` 找口径
2. 写 SQL → 跑 `node scripts/query.mjs "<SQL>"`
3. 给你 **业务结论 + 数据表 + 折叠 SQL**

⚠️ **首次跑 `query.mjs` 慢一点是正常的** — Windows Defender 在扫描 `duckdb_node.node` 这个文件,扫完(约 30 秒-2 分钟)以后就快了。

## 收到新版怎么办

数据负责人通知有新版 zip 时:

1. 把新版 zip 下载到 **`D:\` 根目录**(直接放 D 盘下,不要放子文件夹)
2. 双击 **`D:\chexian-query\update.bat`**
3. 看到"完成"提示 = 升级完成
4. 在 TRAE 里重新打开项目(关闭再打开),试个问题确认正常

如果 TRAE 提示有文件被占用 → 先关闭 TRAE 再双击 `update.bat`。

数据本身**不需要更新**,数据负责人会自动同步到共享盘,你下次查询直接看到的就是最新数据。

## 当前数据时效

```
打开 PowerShell → 进入 D:\chexian-query → 跑:
node scripts/verify.mjs
```

输出最后会有"签单日期 max",那一天就是当前数据截至日。

## 出错了怎么办

| 现象 | 解决 |
|---|---|
| TRAE 不打开项目 | 重启 TRAE,再"打开文件夹" → `D:\chexian-query` |
| 报"找不到 node" | 找 IT 装 Node.js 22 LTS,装完重启 PowerShell |
| 报 "VIEW 加载失败" / "DATA_BASE" | 检查"此电脑"地址栏能否打开 `\\10.120.0.87\chexian-query`,不能就找数据负责人 |
| 报 "SQL syntax error" | 把整段错误回贴给 GLM,让它读 `docs/duckdb-cheatsheet.md` 后重写 |
| GLM 输出指标不对 | 让 GLM 读 `docs/business-rules.md` 第 N 条铁律后重新计算 |
| 第一次跑很慢(>2 分钟) | Defender 扫描中,等就行;之后会很快 |

## ⚠️ 数据保密(重要)

共享盘的数据**未脱敏**,含车架号、车牌号、被保险人性别等敏感字段。

- ❌ **不要**把查询结果(尤其客户清单)截图发到非工作群
- ❌ **不要**把数据导出到本机后再传给项目外的人
- ❌ **不要**问 GLM "列出所有车架号" 之类原始数据查询(GLM 会把数据上传到智谱服务器)
- ✅ **业务结论**(如"赔付率 65%")可正常分享
- ✅ **聚合表**(如机构 × 险种的数字)一般可分享,但不要含个人姓名
