## 一、险类与险别

### 险类(`insurance_type`)
- `交强险` 国家强制
- `商业保险` 自愿购买

> 一个保单可能同时有交强+商业,在数据中**分两行存储**(同 policy_no,不同 insurance_type)。

### 险别组合(`coverage_combination`)
| 值 | 占比 | 含义 |
|---|---|---|
| `单交` | 54.3% | 仅交强险 |
| `交三` | 23.0% | 交强 + 第三者责任 |
| `主全` | 22.7% | 交强 + 商业全险种 |
| `其他` | 0.0% | 特殊组合 |

> "主全"和"交三"都属于商业险,NCD/自主定价系数都适用。"单交"无商业险。

### 是否交商统保(`is_commercial_insure`)
| 值 | 占比 | 含义 |
|---|---|---|
| `单交` | 54.3% | 仅交强险 |
| `套单` | 44.1% | 交强 + 商业一同在我司投保(交商同保) |
| `单商` | 1.6% | 仅商业(交强不在我司) |
| `其他` | 0.0% | 特殊 |

## 三、保费(`premium`)

- 类型:DOUBLE,可为负(批改退费)
- **统计净保费**:`SUM(premium)` — 正负抵消后是实收
- **统计毛保费**:`SUM(ABS(premium))` — 不考虑退费
- **签单 vs 满期**:
  - 签单保费 = `SUM(premium)`,按 `policy_date` 归属
  - 满期保费 = `SUM(premium * earned_ratio)`,按时间已过比例
  - 已赚保费(同满期保费,行业别名)

### 满期比率公式(`earned_ratio`)

```sql
LEAST(1, GREATEST(0,
  date_diff('day', insurance_start_date, LEAST(CURRENT_DATE, insurance_end_date)) / 365.25
)) AS earned_ratio
```

- 保险起期 > 今天 → 0(未生效)
- 保险止期 < 今天 → 1(已满期)
- 中间 → 比例值
- 365.25 是闰年近似(精确版用 `date_diff('year', start, end)` 但 DuckDB 不直接支持,实际用 365.25 即可)

## 四、关键率值公式

### 1. 满期赔付率
```
满期赔付率 = (已决赔款 + 未决赔款) / 满期保费
         = (settled_amount + pending_amount) / earned_premium
```
SQL 示例:
```sql
SELECT
  SUM(c.settled + c.pending) /
  NULLIF(SUM(p.earned_premium), 0) AS earned_loss_ratio
FROM (
  -- 先聚合保单(去重批改)
  SELECT policy_no,
         SUM(premium) AS net_premium,
         SUM(premium * earned_ratio) AS earned_premium
  FROM policy_all
  GROUP BY policy_no
) p
LEFT JOIN (
  -- 再聚合赔案
  SELECT policy_no,
         SUM(settled_amount) AS settled,
         SUM(pending_amount) AS pending
  FROM claims_all GROUP BY policy_no
) c USING (policy_no)
```

### 2. 出险率(必须年化)
```
出险率 = 出险案件数 / 年化满期暴露
年化满期暴露 = SUM(满期保费) / SUM(签单保费) * 保单数 (粗算)
精确版 = SUM(已赚暴露天数) / 365
```
> 直接 `案件数 / 保单数` 是错的,会被未满期保单稀释。

### 3. 综合费用率
```
综合费用率 = 费用金额 / 签单保费
```

### 4. 综合成本率
```
综合成本率 = (已报告赔款 + 费用金额) / 签单保费
        = 满期赔付率 + 综合费用率(粗算)
```

### 5. 续保率(交商同保口径)
```
分母:上一年 insurance_start_date 在期间内 + is_commercial_insure='套单' 的保单
分子:分母中,renewal_policy_no 在本期间被引用的保单数
```
> 摩托和单商不计入分母。摩托续保单独算。

### 6. 推介率(驾意险)
```
分母:商业险出单件数(coverage_combination IN ('主全','交三'),按车架号去重)
分子:驾意险推介件数(cross_sell_all 中 type=驾意)
```

### 7. 渗透率(驾意险)
```
渗透率 = 驾意险承保件数 / 商业险承保件数
```

## 五、NCD 体系

### 商车自主定价系数(`commercial_pricing_factor`)
- 范围 [0.5, 1.5],新能源上限 1.45(2024 起调整)
- 仅**商业险**保单有值,交强险此字段为空
- < 1.0 = 优惠(连续无赔款年份多)
- > 1.0 = 上浮(出险次数多)
- = 1.0 = 基准

### 交强险NCD(`compulsory_ncd` + `compulsory_ncd_factor`)
A0-A6 分组对应官方系数(银保监会 2007 38 号文):
| NCD | 系数 | 含义 |
|---|---|---|
| A0 | 1.0 | 不浮动(新保) |
| A1 | 0.9 | 上一年无赔 |
| A2 | 0.8 | 连续 2 年无赔 |
| A3 | 0.7 | 连续 3 年及以上无赔 |
| A4 | 1.0 | 上一年 1 次有责事故 |
| A5 | 1.1 | 上一年 2 次有责事故 |
| A6 | 1.3 | 上一年发生有责死亡事故 |

> 5 座非营业个人客车基础保费 950 元:A3=665,A6=1235。
> NCD 是监管强制,非公司可控因子。

## 六、批改业务

### 批改类型(`endorsement_type`,仅批改记录有值)
| 类型 | 占比 | 说明 |
|---|---|---|
| `16退保` | 52.9% | 提前退保,产生**负保费** |
| `51车辆过户` | 19.6% | 触发 `is_transfer=True` |
| `42变更车辆基本信息` | 18.7% | 修改发动机号/车架号 |
| `01批改关系人` | 4.7% | 投保人/被保险人变更 |
| `37批改险别信息` | 2.0% | 增减险种 |
| `A5新车上牌` | 1.4% | 临牌转正式牌 |
| `68/69停驶相关` | 0.5% | 保费暂停/恢复 |
| `34/13` | 0.1% | 特别约定/合同解除 |

> 批改记录通过 `endorsement_no = 保单号-001` 关联原始保单。
> **聚合保单数据时**:同一 policy_no 多行(原单+多次批改),`SUM(premium)` 自动抵消。

## 九、摩托车专项口径

- 摩托车几乎全是 `单交`(仅交强险)
- 摩托管理成本通常占 7.3%-13.8%
- 摩托和车险**不直接合并对比**,需独立计算
- A 类机构(摩托业务大):侧重网点开发
- B 类机构(摩托业务小):侧重审车场景捆绑

## 十、特殊处理

- **空值**:`renewal_policy_no` 空 = 非续保;`endorsement_no` 空 = 非批改
- **时间字符串转 DATE**:`CAST(policy_date AS DATE)`,数据已是 DATE 通常不用转
- **车型颗粒过细**(2.3w+ 不同值):分析品牌时 JOIN `dim_brand`
- **车牌归属地**:`SUBSTRING(plate_no, 1, 1)` 是省份字(如"川"),`SUBSTRING(plate_no,1,2)` 是省+市级
